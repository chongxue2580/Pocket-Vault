# 代码风格与约定

- 交流与文档语言：中文优先。
- 代码注释要求：使用中文注释；仅在逻辑不自明时添加简洁注释，避免废话式注释。
- 命名风格：代码标识符建议遵循 Kotlin/Android 社区惯例，类名使用 PascalCase，函数与变量使用 camelCase，常量使用 SCREAMING_SNAKE_CASE。
- 架构约定：采用 MVVM + Clean Architecture，避免把业务逻辑直接写进 Activity/Composable。
- UI 约定：
  - 使用 Jetpack Compose。
  - 优先抽象设计系统组件，避免页面重复堆砌。
  - 保持统一的间距、圆角、字号、动画时长规范。
  - 视觉上避免普通工具化表单风格，强调高级卡片陈列感、柔和阴影、克制动画。
- 产品实现原则：
  - Offline First / Local First
  - Security First（敏感字段必须加密，不明文落库）
  - 轻量、可维护、可扩展
- 数据与安全约定：
  - 密码等敏感字段仅在需要展示时解密。
  - 列表页默认不展示密码明文。
  - 异常处理应避免单条数据异常导致全局崩溃。
- 工程实践建议：
  - Repository / UseCase / ViewModel 职责清晰。
  - 页面应具备空状态、加载状态、异常状态。
  - 功能实现优先组件化、模块化，避免超长文件与巨型类。
- 当前仓库特别说明：
  - 现阶段尚无实际源码，以上约定主要来自 `docs/提示词.md`、`docs/离线收藏盒_app_项目文档.md` 以及仓库级 AGENTS 指引。
