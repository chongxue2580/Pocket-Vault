# 任务完成检查清单

## 通用检查
- 确认改动符合中文文档/注释约定。
- 确认架构边界清晰：UI、Domain、Data、Core 职责未混乱。
- 确认没有把核心业务逻辑直接塞进 Activity 或大型 Composable。
- 确认页面至少具备空状态、异常状态，必要时具备加载状态。
- 确认敏感信息不以明文形式持久化。

## Android 工程建立后建议执行
- 运行 `./gradlew.bat testDebugUnitTest`
- 运行 `./gradlew.bat lintDebug`
- 运行 `./gradlew.bat assembleDebug`

## 产品侧重点
- 离线场景下核心功能可用。
- 首页、搜索、分类、详情、新建/编辑流程可闭环。
- 链接、文字、图片、密码四类内容的数据流一致且可维护。
- 密码查看与解密流程只在必要时触发。
- UI 风格保持统一，避免退化为普通备忘录式界面。

## 当前仓库特别说明
- 仓库目前只有产品文档，没有真实代码与构建系统。
- 因此后续任何开发任务，第一步都应是初始化 Android 工程骨架。
