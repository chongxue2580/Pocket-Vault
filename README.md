# Pocket Vault

Pocket Vault（拾光盒）是一款 **Android 本地离线收藏应用**。  
它不是传统备忘录，而是一个更偏向 **私人收藏匣 / 灵感盒 / 本地密码本** 的产品。

用户可以用它统一保存：

- 网站链接
- 文字片段
- 图片
- 账号密码

整个应用强调：

- 打开即用
- 离线优先
- 本地优先
- 轻量流畅
- 高颜值卡片化界面
- 私密安全

仓库地址：<https://github.com/chongxue2580/Pocket-Vault>

## 产品定位

拾光盒面向的是一类明确的使用场景：

- 刷到一个有趣网站，想立刻收起来
- 看到一句喜欢的话，想马上记住
- 保存一张图片作为灵感或参考
- 记录网站账号、密码和备注
- 在没有网络时也能继续查看与管理内容

它想解决的不是“记事”，而是“收纳”：

- 内容要有收藏感
- 页面要有质感
- 操作要足够短
- 本地数据要可靠
- 敏感信息要安全

## 核心功能

### 1. 四类内容统一收纳

支持以下四种内容类型：

- 链接
- 文字
- 图片
- 账号密码

所有内容都具备统一的基础能力：

- 标题
- 收藏夹
- 标签
- 置顶
- 收藏
- 创建时间
- 更新时间

### 2. 浏览与管理

- 首页内容流
- 分类页筛选
- 搜索页检索
- 收藏夹管理
- 标签管理
- 详情页查看
- 新建 / 编辑 / 删除

### 3. 安全与加密

已支持：

- 主密码
- 二次密码
- 应用锁
- 生物识别开关
- 截图保护
- 普通加密
- 超级加密
- 加密空间

#### 普通加密

- 内容可出现在普通列表中
- 进入详情前需要验证主密码
- 可选“标题公开”或“标题加密”

#### 超级加密

- 内容不出现在普通首页流
- 通过加密空间查看
- 进入加密空间先验证主密码
- 若配置二次密码，则再做一次验证

## 项目结构

当前代码按职责拆分为：

- `app/ui`
- `app/domain`
- `app/data`
- `app/core`

其中：

- `ui` 负责页面、状态与导航
- `domain` 负责模型、仓库接口、用例
- `data` 负责数据库、映射、仓库实现、安全与存储
- `core` 负责通用格式化、设计系统组件与主题

## 本地构建

### 环境要求

- JDK 17+
- Android SDK
- Android Studio 或可用 Gradle 环境

### 当前说明

这个仓库原始工程未携带 `gradlew` / `gradlew.bat` / `gradle-wrapper.jar`。  
如果你要在命令行直接构建，需要先生成 Wrapper：

```bash
gradle wrapper
```

之后可以使用：

```bash
./gradlew assembleDebug
./gradlew testDebugUnitTest
./gradlew lintDebug
```
