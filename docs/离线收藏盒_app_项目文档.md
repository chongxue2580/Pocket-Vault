# 离线收藏盒 App 项目文档（给 Codex 的完整实现说明）

## 1. 项目概述

### 1.1 项目名称
**暂定名：拾光盒 / Pocket Vault / 收藏匣**

建议最终命名风格：
- 中文名：**拾光盒**
- 英文名：**Pocket Vault**

### 1.2 项目定位
这是一个 **Android 本地离线收藏 App（APK）**，用户打开即用，不依赖联网，不需要注册登录，主打：
- 收藏有趣的网站链接
- 收藏喜欢的文字片段
- 收藏照片/图片
- 收藏网站账号密码
- 快速记录、快速检索、快速归类
- 高颜值 UI、精美组件、轻量流畅、私密安全

### 1.3 核心理念
这个 App 不是传统“备忘录”，而是一个更像 **私人收藏匣 + 视觉卡片库 + 离线密码本 + 灵感盒子** 的产品。

关键词：
**离线、轻量、打开即用、漂亮、顺手、安全、归档感、收藏感**

---

## 2. 产品目标

### 2.1 用户痛点
用户当前对传统备忘录不满意，主要问题通常包括：
1. 页面不好看，视觉单调。
2. 收藏内容混在一起，缺乏“收藏品”感。
3. 链接、文字、图片、账号密码的记录方式不统一。
4. 使用流程重，打开慢，功能复杂。
5. 很多应用强依赖联网、同步、账户体系。
6. 密码记录不够私密、不够便捷。

### 2.2 本项目目标
打造一款：
- **无需联网即可完整使用** 的 Android App
- **无需登录，安装即用**
- **收藏效率高**：2~3 步完成一条收藏
- **视觉体验强**：界面高级、卡片精美、动效细腻
- **数据安全**：敏感数据本地加密保存
- **检索方便**：搜索、筛选、标签、分类、置顶、收藏状态完善
- **轻量稳定**：低内存、低功耗、低权限依赖

### 2.3 非目标
本期不做：
- 云同步
- 多设备实时同步
- 社区分享
- 复杂协作
- 联网网页全文抓取
- 自动识别网页元信息（可留作后续扩展）

## 3. 用户画像

### 3.1 核心用户
1. 喜欢收藏有趣链接的人
2. 经常记录短句/文案/灵感的人
3. 会保存图片作为灵感参考的人
4. 想用简单方式管理网站账号密码的人
5. 不喜欢传统备忘录复杂、丑、杂乱的人

### 3.2 使用场景
- 刷手机时看到有趣网站，随手保存链接
- 看到一段喜欢的话，立刻保存
- 保存一张图片作为灵感或回忆
- 记下某网站账号、密码、备注
- 离线状态也能查找全部收藏内容

---

## 4. 产品原则

1. **打开即用**：首次打开直接进入首页，不强制注册，不强制引导。
2. **离线优先**：所有核心功能在无网环境下可用。
3. **输入极简**：新建流程必须足够短。
4. **视觉高级**：像一个设计师做的收藏馆，而不是工具表单。
5. **敏感内容优先保护**：密码类信息默认加密。
6. **轻量克制**：避免功能过载。
7. **可扩展但不臃肿**：底层结构可支持后续升级。

## 7.2 首页

### 7.2.1 页面目标
首页是整个 App 的核心入口，要让用户一进来就感到：
- 内容整齐
- 视觉高级
- 收藏很有质感
- 可以快速继续使用

### 7.2.2 页面模块
1. 顶部问候区
   - App Logo / 名称
   - 搜索入口
   - 今日一句短语（可本地随机静态文案）

2. 快速筛选 Chips
   - 全部
   - 链接
   - 文字
   - 图片
   - 密码
   - 置顶
   - 最近

3. 卡片流内容区
   - 推荐使用 **双列瀑布流 / 自适应网格卡片**
   - 不同类型卡片样式不同

4. 悬浮添加按钮

### 7.2.3 首页卡片设计
#### 链接卡片
- 大标题
- 域名小标签
- URL 精简展示
- 备注摘要
- 彩色渐变背景或玻璃拟态风格

#### 文本卡片
- 大号引文样式
- 支持引号装饰
- 底部可显示来源
- 字体排版要有高级感

#### 图片卡片
- 大图预览
- 可叠加备注角标
- 圆角大图 + 柔和阴影

#### 密码卡片
- 默认不显示密码全文
- 显示站点名、用户名、标签
- 密码区默认遮挡，点击后短时显示
- 强调私密视觉语言

## 8. 视觉设计方案

## 8.1 总体风格关键词
- 高级感
- 玻璃拟态 + 柔和阴影
- 圆角大卡片
- 通透层级
- 舒适留白
- 细腻动画
- 温柔但有辨识度的配色

## 8.2 设计风格建议
推荐主风格：
**现代极简 + 半拟态 + 卡片陈列馆风格**

避免：
- 传统工具类表单感
- 过重商务风
- 过度花哨的渐变堆砌

## 8.3 色彩建议
### 浅色主题
- 背景：奶白 / 雾灰白
- 主色：雾蓝、鼠尾草绿、淡紫
- 强调色：珊瑚橙 / 琥珀黄
- 文本：深灰黑

### 深色主题
- 背景：蓝黑 / 墨灰
- 卡片：深灰 + 轻微透明层
- 强调色：青蓝 / 紫罗兰 / 暖金
- 文本：高亮白 + 次级灰

## 8.4 组件规范
### 按钮
- 大圆角
- 按下有轻微缩放
- 图标与文字间距统一

### 输入框
- 不要传统硬边框
- 使用柔和背景块 + focus 发光边缘
- 密码输入框支持眼睛图标切换

### 卡片
- 圆角 20~28dp
- 阴影柔和
- 内边距舒适
- 支持局部渐变和模糊感

### 标签 Chips
- 胶囊形
- 支持选中态动画
- 颜色统一但可按类型区分

## 8.5 动效规范
- 页面切换：淡入 + 轻微位移
- 卡片点击：缩放至 0.98 再回弹
- FAB 展开：弧线展开或渐显上浮
- 删除操作：滑动消失 + 撤销 Snackbar
- 搜索结果变化：列表平滑过渡

## 10. 技术实现方案（Android）

## 10.1 技术栈建议
推荐使用：
- **语言**：Kotlin
- **UI**：Jetpack Compose
- **架构**：MVVM + Clean Architecture
- **导航**：Navigation Compose
- **本地数据库**：Room
- **依赖注入**：Hilt
- **异步**：Kotlin Coroutines + Flow
- **图片加载**：Coil
- **本地加密**：Android Keystore + Jetpack Security / Tink
- **序列化**：Kotlinx Serialization
- **日志**：Timber（debug 可用）

为什么选 Compose：
1. 更适合做高颜值界面
2. 动效更灵活
3. 组件封装能力强
4. 更利于快速迭代精美 UI

## 10.4 加密方案

### 10.4.1 原则
- 用户密码信息不能明文保存在数据库中
- 加密密钥不直接写死在代码里

### 10.4.2 推荐实现
- 使用 **Android Keystore** 生成并保护主密钥
- 使用该主密钥对敏感字段进行 AES 加密
- 数据库存储密文字符串

### 10.4.3 解密时机
- 只有在详情页需要展示密码时才解密
- 列表页默认不解密全文

### 10.4.4 失败处理
- 密钥异常时给出用户友好提示
- 不要因单条数据异常导致应用崩溃

---

## 10.5 图片存储方案

### 10.5.1 本地文件策略
- 将用户导入图片复制到 App 私有目录
- 保留原图副本
- 生成缩略图用于列表加载

### 10.5.2 原因
- 避免外部路径失效
- 加载更快
- 更利于统一管理与备份

### 10.5.3 注意事项
- 删除 Item 时同步清理文件
- 使用 Coil 加载缩略图
- 大图详情页按需加载

可搜索字段汇总：
- title
- summary
- url
- siteName
- note
- content

- email

## 10.7 权限策略
尽量少要权限：
- 读取图片（根据 Android 版本使用系统照片选择器优先）
- 生物识别权限（如启用）

不需要的权限：
- 通讯录
- 定位
- 麦克风
- 相机（首期不强依赖）

## 18. 异常与边界情况

1. URL 非法：给出校验提示

6. 删除收藏夹时：提示是否迁移其中内容
7. 删除密码条目：二次确认
8. 应用锁忘记 PIN：需设计重置流程（例如清空本地安全配置，但保留非加密内容，或提示风险）
