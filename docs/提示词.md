

# 

你现在是一名资深 Android 产品工程师 + 资深移动端 UI 工程师 + 资深架构师。
请帮我开发一个 **Android 本地离线收藏 App**，输出一个可以运行、可继续迭代、可打包 APK 的完整项目。

------

## 一、项目目标

我要做一个 **手机离线收藏 App**，不是传统备忘录。
它主要用来收藏以下内容：

1. 有趣的网站链接
2. 喜欢的文字/句子/文案
3. 照片/图片
4. 网站账号密码

这个 App 的核心要求：

- **打开即用**
- **不要求联网**
- **不要求登录注册**
- **轻量**
- **本地优先**
- **界面非常漂亮**
- **组件非常精美**
- **高颜值**
- **使用顺手**
- **私密安全**

我不喜欢普通备忘录那种单调页面、普通表单、工具味太重的感觉。
我要的是一种 **像“私人收藏馆 / 私人灵感匣 / 本地密码收藏盒”** 的产品体验。

------

## 二、开发目标

请你直接生成一个完整 Android 项目，要求：

- 使用 **Kotlin**
- 使用 **Jetpack Compose**
- 使用 **MVVM + Clean Architecture**
- 使用 **Room** 作为本地数据库
- 使用 **Hilt** 做依赖注入
- 使用 **Coroutines + Flow**
- 使用 **Android Keystore** 做敏感信息加密
- 支持 **深色模式 / 浅色模式**
- 核心功能必须 **完全离线可用**
- 架构清晰
- 代码可维护
- 页面精美
- 组件化完善
- 项目可以直接运行并继续扩展

请不要做成 Demo 拼凑工程，我要的是 **有明确结构、可继续开发的正式项目骨架 + 首期完整功能实现**。

## 三、产品定位

这是一个 **离线本地收藏 App**，功能上像：

- 链接收藏盒
- 文本灵感本
- 图片收藏册
- 私人密码本

但整体视觉和体验不能像传统工具软件，而应该像：

## 十、加密与安全要求

### 密码存储

- 不允许明文保存密码
- 必须使用 Android Keystore 生成和保护主密钥
- 使用该密钥对密码字段进行加密
- 数据库存储密文

### 解密规则

- 只在查看密码详情时解密
- 列表页默认隐藏密码

### 安全异常处理

- 不能因为某条记录加解密失败导致整个 App 崩溃
- 需要用户可理解的错误提示
